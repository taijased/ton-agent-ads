# Name

summarize-changes

## Purpose

Create a short summary of code or doc changes.

## When to use

- preparing a concise update
- summarizing a finished scope
- converting a diff into a brief changelog-style note

## When not to use

- replacing full design or research docs
- generating a detailed report

## Inputs

- changed files or diff
- optional task context

## Steps

1. identify the main change themes
2. group related file changes
3. write a short factual summary

## Output format

- one short overview
- concise bullets by changed area

## Constraints

- keep output short
- do not repeat full diffs or docs

## Relation to commands/workflows

This skill supports concise chat or handoff summaries.
It does not replace docs or formal reports.
