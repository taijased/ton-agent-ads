# Name

prepare-handoff

## Purpose

Prepare a concise final handoff summary for a completed task.

## When to use

- after implementation or research is complete
- before final user-facing summary

## When not to use

- as a replacement for docs
- during early research or design stages
- for full report generation by default

## Inputs

- changed files
- completed checks
- key outcomes

## Steps

1. identify completed work items
2. list changed areas succinctly
3. include verification status
4. include any remaining assumptions briefly

## Output format

- outcome summary
- changed areas
- verification summary
- remaining notes

## Constraints

- do not dump full artifacts
- keep output short

## Relation to commands/workflows

This skill supports final command output.
It does not replace docs or command-owned handoff structure.
